# Journal Skill — 과정 기록 & 합성

> agent-crew 공유 자산. 프로젝트 고유 값은 `.claude/project.yaml`에서 읽는다.

## 작업 전 — 프로젝트 설정 로드 (필수)

**시작 전 반드시 `.claude/project.yaml`을 Read**하여 다음을 얻는다:

- `project.name` — 프로젝트 식별자
- `knowledge.mode` — `vault`가 아니면 멈추고 보고 (v0.1.0은 vault 모델만)
- `knowledge.vault.path` — 지식 vault 레포 절대 경로
- `knowledge.vault.rawPrefix` — raw 시드 파일 prefix

본문에서 vault 경로는 `{vault.path}`, prefix는 `{rawPrefix}`로 표기한다.

## 배경

결과물뿐 아니라 **"과정"**을 남긴다 — 어떤 사고를 거쳤는지, AI를 어떻게 활용했는지,
AI 산출물에 본인 판단을 어떻게 더했는지. 이 기록이 쌓여 나중에 지식이 된다.

## 이 스킬의 자리 — journal-recorder·wiki-curator와의 경계

| | 무엇을 | 트리거 | 결과물 |
|---|---|---|---|
| **journal 스킬** (이것) | 사용자 대화 맥락 → 시드 / 시드 → 1회성 문서 | 사용자 직접 호출 | raw 시드 · 제출용 문서 |
| **journal-recorder** (에이전트) | 에이전트·작업 산출물 → 시드 | 오케스트레이터 자동 | raw 시드 |
| **wiki-curator** (에이전트) | raw 시드 → wiki 합성 페이지 | 작업 완료 후 | 영구 지식베이스 (복리) |

→ journal 스킬의 합성 모드는 *1회성 산출물*(보고서·과정 기록서 등)을 만든다.
*영구 지식베이스로의 승격*은 wiki-curator 몫이다. 둘 다 raw에서 합성하지만 목적지가 다르다.

## 기록 위치

raw 시드는 journal-recorder와 **같은 위치**에 쌓인다 (한 곳에 모여야 합성이 된다):

```
{vault.path}/raw/{rawPrefix}-<YYYY-MM-DD>-<유형>.md
```

유형: `decisions` · `ai-usage` · `progress`

## 모드 1 — 기록 (Record)

사용자가 "기록해줘"로 호출하거나, 대화 중 의사결정·AI 활용이 감지되면 시드를 append한다.
같은 날짜·유형 파일이 있으면 추가, 없으면 생성한다.

### 의사결정 (`decisions`)
```markdown
## Decision N: {제목}
- **시점**: {언제}
- **선택**: {무엇을 선택}
- **대안**:
  - (A) {대안} — {장단점}
  - (B) {선택한 것} ✅
- **이유**: {구체적 근거}
- **결과**: {결정의 결과}
```
핵심: "선택한 것"만이 아니라 **"대안은 뭐였고 왜 버렸는지"**가 가치 있다.
대안이 1개뿐이면 의사결정이 아니다.

### AI 활용 (`ai-usage`)
```markdown
## AI-N: {작업명}
- **도구**: {사용한 도구}
- **목적**: {AI로 뭘 하려 했나}
- **AI 제안**: {AI가 제안·생성한 내용}
- **본인 판단**: {AI 제안에 본인이 더한 판단}
- **결과**: {최종 결과물}
```
핵심: **"AI 제안"과 "본인 판단"을 분리**한다. "본인 판단"이 비면 안 된다.

### 진행상황 (`progress`)
```markdown
## 수행 내용
- {한 일}

## 기술적 문제 해결
- **문제**: {뭐가 안 됐나}
- **해결**: {어떻게}
- **배운 점**: {알게 된 것}
```

## 모드 2 — 합성 (Generate)

"과정 기록서 만들어줘", "AI 활용 기록 정리해줘" 등으로 호출하면
`{vault.path}/raw/{rawPrefix}-*` 시드를 모아 사용자가 원하는 포맷의 문서를 생성한다.

- 시드를 날짜·유형별로 조합한다
- 시드를 그대로 복붙하지 않는다 — 읽기 좋게 정제하되 핵심 정보(특히 "본인 판단")는 유지
- 생성 위치는 사용자가 지정 (제출 폴더·프로젝트 docs 등). vault raw는 그대로 둔다

> 영구 지식베이스로 남길 내용이면 합성 모드 대신 wiki-curator로 승격하라고 안내한다.

## 사용 흐름

- **작업 중**: 의사결정·AI 활용이 생기면 → 기록 모드로 시드 append
- **하루 끝**: "오늘 기록 정리" → progress 시드 보강
- **산출물 필요 시**: "과정 기록서 생성" → 합성 모드

## 주의사항

- 시드는 **원본 기록** — 수정보다 추가(append)를 우선한다
- 의사결정의 "대안"은 최소 2개
- AI 활용의 "본인 판단"은 비우지 않는다
- 회사 정보(NDA 파트너·민감정보)는 추상화 후 기록한다
