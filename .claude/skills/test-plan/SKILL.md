# Test Plan Skill — 테스트 계획 수립

> agent-crew 공유 자산. 프로젝트·스택 고유 값은 `.claude/project.yaml`·`references/`에서 읽는다.

## 작업 전 — 프로젝트 설정 로드 (필수)

시작 전 `.claude/project.yaml`을 Read하여 `project.name`·`project.stack`을 확인한다.

## 역할

기능·스펙의 **테스트 계획**을 만드는 사용자 진입점이다. 실제 계획 수립은
`test-strategist` 에이전트에 위임한다 — 스킬은 진입점, 에이전트가 워커.

> journal 스킬 ↔ journal-recorder 에이전트와 같은 구도다.

## 흐름

1. **대상 확인** — 무엇의 테스트 계획인가: 기능 설명 / 스펙·PRD 문서 / 작업 이슈 / 코드 범위.
   모호하면 사용자에게 묻는다.
2. **test-strategist 호출** — 대상을 넘겨 계획을 받는다.
3. **계획 전달** — test-strategist의 출력(시나리오 → 피라미드 매핑)을 사용자에게 제시.

## 산출물

test-strategist의 테스트 계획 매니페스트 — 시나리오 목록, 피라미드 레이어 매핑,
스택별 테스트 접근, 커버리지·피라미드 형태 점검.

> 이 스킬은 *계획*까지다. 테스트 코드 작성·실행은 이 계획을 입력으로 하는 별도 작업이다.

## 사용 흐름

- **기능 착수 전** — "이 기능 어떻게 테스트하지" → 계획부터
- **스펙 확정 직후** — notion-doc-writer 스펙 → 곧바로 테스트 계획
- **회귀 점검** — 기존 코드 범위를 대상으로 계획 재수립

## 주의

- 계획은 *피라미드를 누른다* — 가장 싼 레이어 우선
- 스택 특화 권고는 `project.stack`에 달림 — 미설정이면 스택 무관 계획만
